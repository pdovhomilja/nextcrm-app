NextCRM:
Nuestro Proyecto de Pruebas

Una aplicación CRM real y completa donde aplicamos
técnicas de pruebas de software en un entorno de producción.

¿Por qué elegimos NextCRM?

Seleccionamos NextCRM como nuestro proyecto de pruebas porque representa un sistema real, completo y desafiante que nos permite aplicar todo lo aprendido sobre testing de software.

🎯

Aplicación Real y Completa

A diferencia de proyectos académicos o ejemplos de tutorial, NextCRM es una aplicación CRM funcional con múltiples módulos interconectados: facturas, contactos, oportunidades, actividades, auditoría y más. Esto nos permite probar integraciones reales entre componentes.

🔧

Stack Tecnológico Moderno

Construido con Next.js 14, TypeScript, Prisma, PostgreSQL, TailwindCSS y más. Este stack moderno nos permite explorar pruebas en un entorno de desarrollo actual, incluyendo server actions, React Server Components y manejo de datos con ORM.

📊

Datos y Flujos de Negocio Reales

Incluye flujos de trabajo completos: creación de facturas con ciclos de estado, enriquecimiento de datos con IA, búsqueda semántica, y gestión de permisos. Cada flujo representa un caso de prueba valioso.

🧪

Ideal para Pruebas de Integración

La arquitectura con server actions, API routes y MCP server nos permite diseñar pruebas de integración entre frontend, backend y servicios externos (OpenAI, Resend, Inngest) en un único codebase.

¿Por qué son necesarias las pruebas de software?

Las pruebas no son solo una fase del desarrollo — son una inversión en calidad, confianza y mantenibilidad del software.

🛡️

Garantiza Calidad

Un CRM maneja datos críticos del negocio: clientes, facturas, oportunidades. Un error puede costar dinero real. Las pruebas detectan defectos antes de que lleguen a producción, protegiendo la integridad de los datos y la reputación de la empresa.

🔄

Facilita el Mantenimiento

El código de NextCRM evoluciona constantemente con nuevas funcionalidades. Una suite de pruebas bien diseñada permite refactorizar con confianza, agregar features sin miedo a romper lo existente, y documentar el comportamiento esperado del sistema.

⚡

Ahorra Tiempo y Costos

Detectar un bug en producción es 10-100 veces más costoso que encontrarlo durante el desarrollo. Las pruebas automatizadas ejecutan cientos de escenarios en segundos, algo imposible de hacer manualmente en cada release.

🤝

Mejora la Colaboración

Las pruebas sirven como documentación ejecutable. Un nuevo desarrollador puede entender el sistema leyendo los tests. En nuestro grupo de pruebas, nos permite trabajar en paralelo sobre diferentes módulos sin conflictos.

Áreas de Prueba

Las funcionalidades de NextCRM que analizamos y validamos como parte de nuestro proyecto de pruebas.

Módulo de Facturas

NUEVO

Validamos el sistema de facturación completo — flujos de creación, emisión, pago y cancelación. Probamos cálculos de impuestos, multi-moneda y generación de PDF para garantizar precisión.

  • — Tipos de factura: Factura, Nota de Crédito, Proforma y Recibo
  • — Líneas de artículo con cantidad, precio unitario, descuento % y totales automáticos
  • — Motor fiscal configurable (IVA, GST) con desglose por línea
  • — Series de facturas auto-numeradas con prefijo/sufijo
  • — Multi-moneda con formato localizado
  • — Ciclo de estados: BORRADOR → EMITIDA → PAGADA / PARCIALMENTE_PAGADA / CANCELADA
  • — Registro de pagos parciales/totales con seguimiento de saldo
  • — Duplicado y cancelado con un clic, con trazabilidad de auditoría
  • — Envío por email vía Resend con plantillas de React Email
  • — Generación de PDF en el servidor
  • — i18n completo (Inglés y Checo)

Actividades CRM

NUEVO

Probamos el seguimiento de actividades en entidades CRM, validando feeds paginados, vinculación multi-entidad y revalidación en tiempo real tras mutaciones.

  • — Tipos de actividad: Notas, Llamadas, Emails, Reuniones, Tareas
  • — Crear/editar/eliminar inline en cada página de detalle de entidad
  • — Paginación por cursor compuesto para orden estable
  • — Registros vinculados vía crm_ActivityLinks (adjuntos multi-entidad)
  • — Revalidación de ruta en tiempo real tras cada mutación

Registro de Auditoría e Historial

NUEVO

Verificamos el historial completo de cambios por entidad, probando borrado lógico, restauración de registros y diffs JSON estructurados con comparación antes/después.

  • — Línea temporal por entidad de todos los cambios de campos
  • — Borrado lógico con columna deletedAt que preserva datos
  • — Registro de auditoría global filtrable en /admin/audit-log
  • — Restauración de registros eliminados lógicamente
  • — Diffs JSON estructurados con comparación antes/después

Enriquecimiento con IA

NUEVO

Probamos el enriquecimiento con IA vía sandbox E2B, validando puntuación de confianza, cifrado AES-256-GCM de claves y degradación elegante ante fallos.

  • — Sandbox Linux completo con navegador Chrome real
  • — Claude Sonnet 4.6 impulsa la investigación con bucle de herramientas
  • — Descubrimiento de contactos C-level desde solo el nombre de empresa
  • — Estrategia consciente del contexto omitiendo investigación innecesaria
  • — Puntuación de confianza (descartado por debajo de 0.6)
  • — Prioridad de claves API en 3 niveles: ENV → Admin → Perfil de usuario
  • — Cifrado AES-256-GCM en reposo para todas las claves
  • — Degradación elegante con diálogos informativos

Servidor MCP

NUEVO

Validamos el servidor MCP con 127 herramientas en 15 módulos, probando autenticación Bearer, transporte HTTP streamable y compatibilidad SSE legacy.

  • — 127 herramientas: Cuentas, Contactos, Leads, Oportunidades, Targets, Productos, Contratos, Actividades, Documentos, Listas de Targets, Enriquecimiento, Cuentas de Email, Campañas, Proyectos, Reportes
  • — Tokens Bearer con hash SHA-256 (nxtc__...)
  • — Transporte HTTP streamable (default del spec MCP)
  • — Soporte de transporte SSE legacy
  • — SKILL.md para Claude Code incluido

Búsqueda Vectorial

NUEVO

Probamos búsqueda semántica con embeddings OpenAI, índices HNSW para ANN rápida y backfill de registros existentes.

  • — OpenAI text-embedding-3-small vía jobs de Inngest
  • — Resultados unificados de búsqueda por palabra clave + semántica
  • — Botón "Encontrar Similares" en cada página de detalle
  • — Función de backfill para registros existentes
  • — Auto-embed al crear/actualizar
  • — Índices HNSW para búsqueda ANN rápida
